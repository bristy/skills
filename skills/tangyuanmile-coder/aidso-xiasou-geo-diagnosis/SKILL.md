# 品牌 GEO 诊断报告（单接口轮询版）

当用户请求类似以下表达时，优先使用这个 skill：
- 查一下 某品牌 的 GEO 表现
- 给我一份 某品牌 的 GEO 报告
- 看看某品牌在 AI 搜索里的可见度趋势
- 帮我做一个某品牌的 GEO 诊断报告
- 帮我看一下某品牌的 GEO 诊断
- 想看某品牌在 AI 搜索里的曝光表现

## 目标
当用户发起某品牌 GEO 诊断请求时，先提示本次诊断将消耗 20 积分，等待用户确认；用户确认后，立刻返回处理中提示；然后再轮询动态接口获取该品牌诊断报告；一旦接口返回 markdown 报告正文，则停止轮询并把完整 markdown 文本内容直接返回给用户。

---

## 动态接口说明
获取品牌诊断报告使用的是动态接口：

```bash
GET "https://testapi.aidso.com/openapi/skills/band_report/md?brandName=品牌名称"
```

说明：
- 固定接口地址：
  `https://testapi.aidso.com/openapi/skills/band_report/md`
- 动态参数：
  - `brandName`: 品牌名称（必填，最多50字符）

也就是说：
- 用户输入什么品牌名
- skill 就提取该品牌名
- 然后把品牌名作为 `brandName` 参数传给接口请求报告

例如：
- 用户请求露露 → 请求：
  `GET "https://testapi.aidso.com/openapi/skills/band_report/md?brandName=露露"`
- 用户请求 Tesla → 请求：
  `GET "https://testapi.aidso.com/openapi/skills/band_report/md?brandName=Tesla"`

---

## Step 1：提取品牌名
从用户输入中提取品牌名称。

规则：
1. 只提取用户明确提供的品牌名。
2. 如果未明确提供品牌名，只问一句：`请输入需要诊断的品牌名称。`
3. 不要擅自补充品牌别名、英文名、集团名或产品线，除非用户明确说明。

---

## Step 2：积分确认
当品牌名已明确，且当前会话中还没有用户对本次诊断的明确确认时，直接回复：

`此次诊断将消耗20积分，是否确认？`

### 重要约束
- 在用户确认前，不要调用接口。
- 不要跳过积分确认提示。

---

## Step 3：处理用户确认
如果用户回复以下含义之一：
- `确认`
- `确认诊断`
- `确认开始`
- `开始诊断`
- `好的`
- `是`
- `yes`

则视为用户已确认本次诊断。

如果用户表示拒绝，例如：
- `取消`
- `不确认`
- `暂不`
- `不要了`

则结束流程，不要调用接口。

---

## Step 4：用户确认后，立刻返回处理中提示
只要用户确认，就先立即向用户返回这句提示：

`正在为「{brandName}」诊断中，大约需要3~10分钟，请稍后；`

### 重要约束
- 这句提示要先返回给用户。
- 不要等接口返回“处理中”后再说。
- 只要用户确认，就立刻说这句。

---

## Step 5：返回处理中提示后，再轮询接口获取报告
在返回处理中提示之后，再执行：

```bash
python3 "{baseDir}/run.py" poll_report "{brandName}"
```

脚本会：
1. 将品牌名作为 `brandName` 参数，动态请求接口：
   `GET "https://testapi.aidso.com/openapi/skills/band_report/md?brandName=品牌名称"`
2. 如果接口返回的是：`正在处理中，请稍后`
   则继续轮询
3. 如果接口返回的是 markdown 报告正文，则立即停止轮询，并直接返回全部 markdown 文本内容
4. 如果接口返回的是文件或文件链接，则优先按文件方式返回

### 重要约束
- 只要已经拿到报告内容，就不要再继续轮询。
- 如果接口返回的是 markdown 文本，不要摘要，不要裁剪，不要保存成 `.md` 文件，不要作为附件返回，直接返回全部 markdown 文本内容。
- 如果接口仍在处理中，继续轮询，不要重复向用户刷“处理中”提示。

---

## Step 6：返回结果
- 如果脚本输出的是普通文本，则原样返回。
- 如果脚本输出的是 `MEDIA:` 行，则原样返回这些 `MEDIA:` 行给用户，并可配一句简短说明，例如：`诊断报告已生成，请查收附件。`

---

## 失败处理
- 如果接口调用失败：简洁说明失败原因，不要编造报告内容。
- 如果品牌名缺失：只追问品牌名，不要问多余问题。
- 如果轮询超时：说明任务仍在处理中，请稍后再试。

---

## 关键约束
- 用户未确认积分前，不要调用接口。
- 如果接口返回的是 markdown 文本，不要保存成 `.md` 文件，不要作为附件返回，直接返回全部 markdown 文本内容。
